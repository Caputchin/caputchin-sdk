Graceful-degradation edge cases

invisible × click → coerced to auto + error event

mode=invisibletrigger=click ✗

Click trigger requires a clickable surface; invisible has none. Config coerces trigger → auto and fires an invalid-config error event. Widget still verifies.

<caputchin-widget sitekey="cpt_pub_WVQVqTdVl92Sh_bxokoz_wCw" mode="invisible" trigger="click"></caputchin-widget>

simple without sitekey → widget inert + error event

mode=simplesitekey ✗

Missing sitekey on a verification mode → widget does not activate. Single invalid-config error event fires.

<caputchin-widget mode="simple"></caputchin-widget>
(no UI; inert)

unknown mode → falls back to "simple" + error event

mode=bogus ✗

Unknown mode value coerces to "simple" with an invalid-config error event.

<caputchin-widget sitekey="cpt_pub_WVQVqTdVl92Sh_bxokoz_wCw" mode="bogus"></caputchin-widget>

game × auto — game calls bridge.error → widget fires game-error-relayed

mode=gametrigger=autobridge.error

The game waits 1.5s then calls bridge.error({code:'demo-failure'}). Widget translates to game-error-relayed with originalCode carrying the game-supplied code.

<caputchin-widget sitekey="cpt_pub_..." mode="game" game="@demo/failing" game-src="/examples/failing-game.js"></caputchin-widget>

game × auto — multi-round passes

mode=gametrigger=autoreplay

Click the button 3 times. Each click fires bridge.pass({score, durationMs}). First call mints the wrapped token and emits the pass event; subsequent calls emit pass events with later scores but the token is locked at the first.

<caputchin-widget sitekey="cpt_pub_..." mode="game" game="@demo/multi-round" game-src="/examples/multi-round-game.js"></caputchin-widget>

simple × form-submit with invalid sitekey → red error state on the pill

mode=simpletrigger=form-submitsitekey ✗

Platform rejects the sitekey on /verify/start. Cap solve throws, the widget fires a verification-failed event and the pill flips to the red error state ("! verification failed"). Same path triggers for: rate limiting, network drop mid-flow, cap-misconfigured, etc. 

<form><caputchin-widget sitekey="cpt_pub_bogus_invalid" trigger="form-submit"></caputchin-widget><button>Submit</button></form>
