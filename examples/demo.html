Caputchin Widget — Dev Demo

All widgets use sitekey="cpt_pub_demo". Platform routes are stubbed 501 today — expect cap-solve-failed errors in the log.

1 — Invisible (auto mode, no game)

Cap-only PoW verification runs on mount. No game UI shown.

2 — form-submit mode

Widget arms on the parent form. Verification runs when the form submits.

3 — manual mode

start() triggers verification. pass() releases the gate early. setNickname() is post-MVP and throws.

4 — customer-hosted game (game-src)

The widget loads examples/simple-game.js in an opaque-origin iframe. Click the button three times to call bridge.pass().

5 — game-only mode

Cap is disabled entirely. The widget hosts the game iframe and surfaces start / pass events with token: null. No network requests to /game/start or /game/pass.

6 — game-only via sitekey-absent shorthand

Same as #5 but no sitekey attribute. Omitting sitekey is shorthand for mode="game-only".
